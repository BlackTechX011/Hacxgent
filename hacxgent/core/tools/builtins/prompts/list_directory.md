# list_directory

List files and directories in a given path with metadata.

## Usage

Use `list_directory` for **exploring the project structure**:
1. Get a clean, structured view of any directory.
2. View file sizes and modification times to identify relevant files.
3. Preferred over bash `ls` for consistent cross-platform behavior.

## Parameters

- `path`: Path to the directory to list (default ".").

## Example

```json
{
  "path": "src/core"
}
```
